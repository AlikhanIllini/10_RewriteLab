**Challenges:**
- Understanding when to use each view type

**Reflections on View Types:**
- HttpResponse is useful for simple responses or API endpoints returning non-HTML

- Base CBV (View) is good when you need custom logic but want class organization
- Generic CBVs are best for standard CRUD operations - less code, more conventions

**Why get_absolute_url() is useful:**
The `get_absolute_url()` method provides a canonical way to get the URL for a model instance. Benefits include:
1. Templates can use `{{ object.get_absolute_url }}` instead of `{% url 'app:view' object.pk %}`
2. Centralized URL logic - if URL pattern changes, only update one place
3. Used by Django admin's "View on site" feature
4. Enables the model to be self-documenting about its primary URL

**BytesIO Memory Awareness:**
- BytesIO holds image data in RAM temporarily
- Buffer is written directly to HttpResponse
- Figure is closed after saving to free matplotlib memory
- For large-scale apps, consider caching generated images

**HttpResponse vs JsonResponse:**
- HttpResponse: Generic response, must set content_type manually
- JsonResponse: Automatically serializes to JSON, sets application/json content type
- Demo at /api/demo/?format=json or ?format=html
