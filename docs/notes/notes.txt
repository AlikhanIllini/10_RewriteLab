### Week 2 (Project Foundations + Views + Templates)
**Date:** February 2026

**Completed:**
- Implemented split settings pattern (base.py, development.py, production.py)
- Configured environment variables with python-dotenv
- Created .env and .env.example files for secure key management
- Set up proper .gitignore to exclude sensitive files
- Restructured docs folder: wireframes, branching_strategy, notes
- Implemented 4 types of views:
  1. FBV with HttpResponse (manual template loading)
  2. FBV with render() shortcut
  3. CBV inheriting from View
  4. Generic CBV (ListView)
- Created base template with block inheritance
- Implemented list template with {% for %} and {% empty %}

**Notes for Grading:**
- View 1 (HttpResponse FBV): `session_manual` - demonstrates manual HTML/template loading
- View 2 (render FBV): `session_list_render` - uses render() shortcut with context
- View 3 (Base CBV): `SessionBaseView` - inherits from View, implements get()
- View 4 (Generic CBV): `SessionListView` - uses ListView with model and template

**Challenges:**
- Understanding when to use each view type
- Template inheritance required careful planning of block structure

**Reflections on View Types:**
- HttpResponse is useful for simple responses or API endpoints returning non-HTML
- render() is the go-to for most template-based views - cleaner and more Pythonic
- Base CBV (View) is good when you need custom logic but want class organization
- Generic CBVs are best for standard CRUD operations - less code, more conventions
